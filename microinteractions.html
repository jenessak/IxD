Like Button

A like button gives users a way to express positive feedback. The key interaction is the toggle — liked / not liked — often with a count and a micro-animation to signal the state change.

Variant 01
Pill Outline
Bordered pill shape with heart icon and live count. Clear state difference between liked and default via fill + color change.
Variant 02
Ghost / Text
No border, no background. Minimal footprint — common in comment threads and social feeds where space is tight.
Variant 03
Filled Background
Solid background that shifts color on activation. Feels tactile and substantial — good for card-based layouts.
Variant 04
Icon + Count Separated
1.2k
likes
Icon and count live separately. Emphasizes the number — good for content with large engagement metrics.
Variant 05
Dark Theme
reactions
Same pill pattern adapted for dark UI. Border switches from dark-gray to red on interaction. Color usage is minimal.
Variant 06
Burst / Ripple
Click me!
Adds a circular ripple that expands outward on activation. Makes the interaction feel more rewarding and physical.
Toggle Switch

Toggles represent an immediate binary setting — on or off. Unlike a checkbox, they imply instant effect without a save action. Key design factors: track, knob, color, and motion.

Variant 01
Classic Round
The standard toggle: rounded track, circular knob, blue when on. Recognizable and platform-safe. Used by iOS, Android, and most design systems.
Variant 02
Icon Indicator
Checkmark and X icons inside the track give an extra visual cue beyond color. Useful for accessibility and colorblind users.
Variant 03
Square / Boxy
Rectangular track with squared corners. Feels more technical and precise — suits data-heavy dashboards, dev tools, or industrial interfaces.
Variant 04
Outlined / Ghost
Transparent fill with a visible border. The knob and border both change color on activation. Light and airy — works well on colored or textured backgrounds.
Variant 05
Jumbo / Large
Larger target area for touch-first interfaces, accessibility needs, or prominent settings. Same mechanics, bigger presence.
Variant 06
With Text Label
Off
The toggle updates a live text label showing current state. Eliminates ambiguity — users know immediately what "on" means without inferring from color.
Form Error

Error states communicate what went wrong and how to fix it. Good form errors are specific, non-alarming, and placed close to the problem. These are six common patterns.

Variant 01
Inline Message
Enter a valid email address.
Error text appears directly below the field. Simple and direct — the most common pattern. Red border reinforces the field is invalid.
Variant 02
Tooltip Bubble
Must be at least 3 characters
Error floats as a tooltip below the input. Saves vertical space — good when fields are densely packed. The arrow anchors it visually to the field.
Variant 03
Icon in Field
Numbers only, no letters.
Alert icon inside the input's right edge gives an immediate cue without disrupting layout. Often combined with a short message below.
Variant 04
Shake Animation
Incorrect password.
The field physically shakes left-right on error. A kinesthetic signal — feels like a "no." Especially effective for wrong-password scenarios.
Variant 05
Banner / Alert Box
Submission failed
Last name is required to continue.
A full-width alert block summarizes the issue. Useful for multi-field forms where errors are caught on submit rather than in real-time.
Variant 06
Error vs Success
This username is already taken.
Username is available!
Showing error and success states side by side clarifies the contrast. Green confirmation reduces anxiety and encourages users to keep going.
