Web Accessibility: Before & After

An Educational Demonstration of Common WCAG Violations and How to Fix Them

About This Demo

This interactive demonstration shows common web accessibility violations side-by-side with their corrected versions. Each example illustrates issues that prevent people with disabilities from accessing web content and demonstrates best practices for creating inclusive websites that comply with WCAG 2.1 AA standards.

Note: The "Before" examples intentionally contain accessibility violations for educational purposes only.

BEFORE Non-Compliant

❌ Accessibility Issues:

  • Color contrast ratio of 1.8:1 (fails WCAG AA 4.5:1 requirement)
  • Text is difficult to read for users with low vision
  • Unusable for people with color blindness

Low Contrast Text Example

This text has very poor contrast against the background. Users with visual impairments, older users, and anyone in bright sunlight will struggle to read this content. The contrast ratio is approximately 1.8:1, which fails WCAG AA standards that require at least 4.5:1 for normal text.

AFTER Compliant

✓ Accessibility Fixes:

  • Color contrast ratio of 16:1 (exceeds WCAG AA requirements)
  • Text is clearly readable for all users
  • Accessible to people with various visual impairments

Good Contrast Text Example

This text has excellent contrast against the background. Users with visual impairments, older users, and anyone in bright sunlight can easily read this content. The contrast ratio is approximately 16:1, which exceeds WCAG AA standards that require at least 4.5:1 for normal text.

Key Takeaways

Essential WCAG 2.1 AA Requirements:

  • Perceivable: Content must be presented in ways all users can perceive (sufficient contrast, alt text, captions)
  • Operable: Users must be able to operate the interface (keyboard navigation, sufficient time, no seizure triggers)
  • Understandable: Information and interface operation must be understandable (clear labels, error identification, consistent navigation)
  • Robust: Content must work with current and future technologies (valid HTML, ARIA attributes, semantic markup)

Quick Compliance Checklist:

  • ✓ Text contrast ratio of at least 4.5:1 for normal text, 3:1 for large text
  • ✓ All form inputs have associated labels
  • ✓ All images have appropriate alt text (or alt="" for decorative images)
  • ✓ Logical heading structure without skipping levels
  • ✓ All functionality available via keyboard
  • ✓ Clear focus indicators on interactive elements
  • ✓ Descriptive link text (not "click here")
  • ✓ Data tables use proper markup with headers
  • ✓ Touch targets at least 44x44 pixels
  • ✓ No content flashes more than 3 times per second
